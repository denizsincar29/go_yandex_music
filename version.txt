## Version 2.0 - Comprehensive Feature Update

### 🎵 Progressive Web App (PR #2)
- Transformed CLI-only player into full-featured PWA
- Modern responsive web interface (desktop, tablet, mobile)
- Installable as standalone app on all platforms
- Service worker for offline caching
- PWA manifest for native app-like experience
- Native HTML5 audio player
- Full WCAG 2.1 AA accessibility compliance
- ARIA labels and semantic HTML structure
- Complete keyboard navigation support

### 🏗️ Architecture Improvements (PR #3)
- Restructured code into cmd/ and internal/ directories
- Separated CLI and web applications properly
- CLI in cmd/cli/ (requires audio libraries)
- Web server in cmd/web/ (no special dependencies)
- Shared code in internal/common/ package
- Standard Go project layout following best practices

### 🔍 Enhanced Search & Navigation (PR #4)
- Album and artist search alongside tracks
- Click-to-navigate album/artist tracks
- Fixed download functionality (proper file download, not browser player)
- Spelling correction feedback
- Semantic headers for screen reader navigation
- Media hardware key support (next/previous)
- Deep linking with URL parameters

### 🐛 Bug Fixes & Stability (PRs #5-9)
- Fixed back button navigation from albums (PR #5)
- Fixed CORS policy errors for external images (PR #6)
- Fixed OpaqueResponseBlocking errors (PR #7)
- Fixed integration tests with proper URL construction (PR #8)
- Fixed album tracks endpoint to use direct API call (PR #9)
- Service worker improvements for external image handling

### 🎨 UI & UX Enhancements (PR #10)
- Systemd service template creator script
- Updated branding (Yandex Music Player title)
- Creator attribution footer
- URL parameter support for deep linking (search and album autoplay)
- Enhanced release automation with auto-changelog generation

### 🚀 CI/CD & Release (PR #11)
- Fixed GitHub Actions workflow failure
- Updated all actions to latest versions (checkout v5, setup-go v6, github-script v8, goreleaser-action v8)
- Separate CLI and web binary assets in releases
- Web binaries for Windows, macOS, and Linux
- CLI binaries for Linux with ALSA support
- Enhanced release automation

### 📝 Testing Infrastructure (PR #7-8)
- Comprehensive unit tests for web server handlers (10 tests)
- Integration tests for search and album functionality
- Test documentation (README_TESTS.md)
- Fixed .env loading for tests
- Multi-page search for better album track results
